Space Invaders: Arcade Classic

Relive the 1978 phenomenon on the Raspberry Pi Pico W. A deep dive into retro graphics, collision logic, and real-time MicroPython gameplay.

Hardware Connections

I2C OLED (SSD1306)

  • VCC → 3.3V
  • GND → GND
  • SDA → GP4
  • SCL → GP5

Control Buttons

  • Left Button → GP14
  • Right Button → GP15
  • Shoot Button → GP16
  • (All buttons connected to GND)
main.py 
from machine import Pin, I2C
import ssd1306
import time
import random

# OLED setup
i2c = I2C(0, sda=Pin(4), scl=Pin(5), freq=400000)
oled = ssd1306.SSD1306_I2C(128, 64, i2c)

# Buttons
left_btn = Pin(14, Pin.IN, Pin.PULL_UP)
right_btn = Pin(15, Pin.IN, Pin.PULL_UP)
shoot_btn = Pin(16, Pin.IN, Pin.PULL_UP)

# Game variables
player_x = 60
player_y = 58
speed = 3

bullet_active = False
bullet_x = 0
bullet_y = 0

score = 0
game_over = False

# Create enemies (2 rows)
enemies = []
for row in range(2):
    for col in range(5):
        enemies.append([10 + col*22, 10 + row*12])

enemy_dir = 1

def reset_game():
    global player_x, bullet_active, enemies, score, game_over, enemy_dir
    
    player_x = 60
    bullet_active = False
    score = 0
    game_over = False
    enemy_dir = 1
    
    enemies.clear()
    for row in range(2):
        for col in range(5):
            enemies.append([10 + col*22, 10 + row*12])

# Main loop
while True:
    oled.fill(0)

    if not game_over:
        # Move player
        if left_btn.value() == 0:
            player_x -= speed
            if player_x < 0:
                player_x = 0

        if right_btn.value() == 0:
            player_x += speed
            if player_x > 120:
                player_x = 120

        # Shoot
        if shoot_btn.value() == 0 and not bullet_active:
            bullet_x = player_x + 3
            bullet_y = player_y
            bullet_active = True

        # Move bullet
        if bullet_active:
            bullet_y -= 4
            if bullet_y < 0:
                bullet_active = False

        # Move enemies
        for e in enemies:
            e[0] += enemy_dir

        # Bounce + move down
        if enemies and (enemies[0][0] < 0 or enemies[-1][0] > 120):
            enemy_dir *= -1
            for e in enemies:
                e[1] += 5

        # Collision detection
        for e in enemies:
            if bullet_active:
                if abs(bullet_x - e[0]) < 6 and abs(bullet_y - e[1]) < 6:
                    enemies.remove(e)
                    bullet_active = False
                    score += 10
                    break

        # Check game over
        for e in enemies:
            if e[1] > 50:
                game_over = True

        # Draw player
        oled.fill_rect(player_x, player_y, 8, 4, 1)

        # Draw enemies
        for e in enemies:
            oled.fill_rect(e[0], e[1], 6, 4, 1)

        # Draw bullet
        if bullet_active:
            oled.pixel(bullet_x, bullet_y, 1)

        # Draw score
        oled.text("Score: {}".format(score), 0, 0)

        # Win condition
        if not enemies:
            oled.text("YOU WIN!", 30, 30)
            game_over = True

    else:
        oled.text("GAME OVER", 25, 20)
        oled.text("Score: {}".format(score), 25, 35)
        oled.text("Press SHOOT", 15, 50)

        # Restart
        if shoot_btn.value() == 0:
            time.sleep(0.3)
            reset_game()

    oled.show()
    time.sleep(0.03)

Technical Analysis

1. Initialization & Assets

The code begins by configuring the I2C bus at 400kHz for the SSD1306 display. Buttons are initialized with PULL_UP resistors, meaning they read 0 when pressed. The enemies are stored as a list of lists, where each sub-list [x, y] represents a single invader.

2. Movement & Boundary Logic

The player is constrained between coordinates 0 and 120 to stay on screen. The enemies move horizontally using an enemy_dir variable. When any invader hits the screen edge, the direction is flipped (multiplied by -1) and the entire row drops down 5 pixels, increasing the difficulty.

3. Collision Detection

Collision is calculated using a simple 2D bounding box method. If the absolute difference between the bullet coordinates and an enemy's coordinates is less than 6 pixels, a hit is registered. The enemy is removed from the list, the bullet is deactivated, and the score increments.

4. Rendering Pipeline

The game uses a "Clear-Update-Draw-Show" loop.

  • oled.fill(0) clears the previous frame.
  • Rectangles and pixels represent the player and projectiles.
  • oled.show() pushes the buffer to the physical screen.
  • time.sleep(0.03) maintains a smooth framerate of ~30 FPS.
