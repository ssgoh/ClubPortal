main.py 
import network
import time
import dht
import urequests
from machine import Pin

# --- Configuration ---
WIFI_SSID = "Your_WiFi_SSID"
WIFI_PASS = "Your_WiFi_Password"
THINGSPEAK_API_KEY = "Your_Write_API_Key"
DHT_PIN = 16 # GP16

# --- LED Setup ---
led_pwr = Pin(15, Pin.OUT)   # Red: Power (GP15)
led_wifi = Pin(14, Pin.OUT)  # Green: WiFi (GP14)
led_data = Pin(13, Pin.OUT)  # Blue: Data (GP13)

# Power LED ON
led_pwr.value(1)

# --- Setup DHT Sensor ---
sensor = dht.DHT11(Pin(DHT_PIN))

# --- Connect to Wi-Fi ---
wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(WIFI_SSID, WIFI_PASS)

while not wlan.isconnected():
    led_wifi.toggle() 
    time.sleep(0.5)
    
led_wifi.value(1) # WiFi connected!

while True:
    # 1. Read the weather
    sensor.measure()
    temp = sensor.temperature()
    hum = sensor.humidity()
    
    # 2. Build the link
    url = f"https://api.thingspeak.com/update?api_key={THINGSPEAK_API_KEY}&field1={temp}&field2={hum}"
    
    # 3. Send to web and blink Blue LED
    led_data.value(1)
    response = urequests.get(url)
    response.close()
    led_data.value(0)
    
    # 4. Wait for next update
    time.sleep(20)

Code Breakdown

1. The Imports

We start by bringing in "tools" (libraries). network handles WiFi, dht reads the sensor, and machine.Pin lets us control the hardware pins.

2. Pin Definitions

We tell Python which pin does what. For example, Pin(15, Pin.OUT) sets GP15 as an output (sending power out) for the Red LED.

3. WiFi Connection

The while not wlan.isconnected() loop is smart—it keeps blinking the Green LED until the Pico successfully grabs a signal from your router.

4. Reading & Sending

This runs forever in a loop. It asks the DHT11 for the weather, builds a web address containing that data, and sends it to ThingSpeak. The Blue LED blinks during the send so you can see the activity!

Lesson Note: We kept this code simple. In future lessons, we will learn how to add "Error Catching" so the program doesn't stop if the WiFi signal is weak.
