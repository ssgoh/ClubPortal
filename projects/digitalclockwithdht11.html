Upload this to your Pico W using Thonny (requires ssd1306.py library):

from machine import Pin, I2C
import ssd1306
import dht
import time

# Pin setup
i2c = I2C(0, sda=Pin(16), scl=Pin(17))
oled = ssd1306.SSD1306_I2C(128, 64, i2c)
sensor = dht.DHT11(Pin(15))

while True:
    try:
        sensor.measure()
        temp = sensor.temperature()
        
        # Get local time from the internal RTC
        t = time.localtime()
        months = ["Jan", "Feb", "Mar", "Apr", "May", "Jun", 
                  "Jul", "Aug", "Sep", "Oct", "Nov", "Dec"]
        
        # Using f-strings for cleaner string formatting
        date_str = f"{t[2]:02d} {months[t[1]-1]} {t[0]%100:02d}"
        time_str = f"{t[3]:02d}:{t[4]:02d}:{t[5]:02d}"
        
        oled.fill(0)
        oled.text(date_str, 25, 10)
        oled.text(time_str, 20, 30)
        oled.text(f"Temp: {temp} C", 25, 50)
        oled.show()
        
    except Exception as e:
        # If the sensor fails (e.g. loose wire), show an error 
        # instead of crashing the whole program.
        oled.fill(0)
        oled.text("Sensor Error", 10, 30)
        oled.show()
        
    time.sleep(1)

Why Use Try/Except?

In hardware projects, sensors like the DHT11 are prone to intermittent communication failures. If a wire is slightly loose or the signal timing is off, the sensor.measure() command will throw an Exception.

  • Prevent Crashing: Without try/except, any small sensor error would stop the entire MicroPython program immediately.
  • Graceful Failure: Using except allows the code to catch the error, display "Sensor Error" on the OLED, and try again in the next loop without stopping.
  • Debugging: It helps you identify if hardware is disconnected while keeping the time-display part of your project running.

Date & Time Logic

The Pico W tracks time using an internal Real-Time Clock (RTC). We access this via time.localtime(), which returns a Python Tuple.

# time.localtime() returns:
t = (2024, 3, 15, 14, 30, 05, 4, 75, 0)
index: [0] [1] [2] [3] [4] [5] [6] [7] [8]

Because Python indices start at 0, we use the following mapping:

  • t[0]: Year (e.g., 2024)
  • t[1]: Month (1 to 12). In the code, we use months[t[1]-1] to get the name.
  • t[2]: Day of the month (1 to 31).
  • t[3], t[4], t[5]: Hours, Minutes, and Seconds.

I2C Communication

The SSD1306 OLED display communicates via I2C. We use GP16 (SDA) and GP17 (SCL). These pins are an alternative set of hardware I2C0 pins on the Pico W, which can help avoid wiring congestion near the top pins.

DHT11 Logic

The DHT11 uses a single-wire digital protocol on GP15. The dht library handles the precise timing required to pull data from the sensor's internal humidity and temperature registers.
