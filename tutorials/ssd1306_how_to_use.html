SSD1306 OLED Display

Mastering the 128x64 I2C display with Raspberry Pi Pico W

Hardware Connection

The SSD1306 communicates via I2C. Connect your OLED to the Pico W using the following pins:

OLED Pin Pico Pin
GND Pin 38 (GND)
VCC Pin 36 (3V3)
SCL GP17 (SCL)
SDA GP16 (SDA)

Note: Pin numbers refer to the physical layout of the Pico W board.

Installing the Library

1

Open Thonny IDE

Ensure your Pico W is connected and the interpreter is set to "MicroPython (Raspberry Pi Pico)".

2

Tools > Manage Packages

In the top menu, go to Tools and click on 'Manage Packages'.

3

Search for "ssd1306"

Type micropython-ssd1306 into the search bar and press Enter.

4

Install

Select the result and click 'Install'. This will automatically save ssd1306.py to your Pico's lib folder.

MicroPython Script

from machine import Pin, I2C
import ssd1306

# Initialize I2C (using GP16 for SDA, GP17 for SCL)
i2c = I2C(0, sda=Pin(16), scl=Pin(17), freq=400000)

# Screen dimensions
WIDTH = 128
HEIGHT = 64

# Initialize OLED
oled = ssd1306.SSD1306_I2C(WIDTH, HEIGHT, i2c)

# Clear the screen
oled.fill(0)

# Write to the display
# Syntax: text(string, x, y)
oled.text("Hello World", 0, 0)      # Row 1 (Top)
oled.text("1234567890", 0, 16)     # Row 2 (Spaced down)

# Send data to physical screen
oled.show()

Understanding 128x64 Pixels

The Grid

The display is a matrix of 128 horizontal pixels (X) and 64 vertical pixels (Y).

Top-Left: 0, 0
Bottom-Right: 127, 63

Text Spacing

The standard font in the library is roughly 8 pixels tall.

To prevent overlapping, we space rows by at least 8-10 pixels. In our example, using y=16 places the second row clearly below the first.

Pro Tip: Always call oled.show() after writing text. The MicroPython library writes to a "buffer" in RAM first; show() is what pushes that memory to the actual hardware.
